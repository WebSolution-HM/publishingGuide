🎯 Publishing 작성 가이드

1. 목적

일관된 형식으로 작성하여 유지보수를 간단하게 하기 위함.

2. 클래스 네이밍 규칙

  • 캐밥 스타일
  • BEM 방식 + 하이픈 표기 header-action--mail, header-action--logout
  • 공통 스타일은 일반적인 선택자 혹은:is([class^="name--"], [class*=" name--"])선택자로
<div class="header-actions">
  <button class="header-action--mail" type="button" aria-label="메일함 열기"></button>
  <button class="header-action--logout" type="button" aria-label="로그아웃"></button>
</div>

3. SCSS 구조

              
  .header-action {
    display: inline-flex;
    align-items: center;
    justify-content: center;
    width: 36px;
    height: 36px;
    border-radius: 8px;
    background-color: var(--header-btn-bg);
    background-repeat: no-repeat;
    background-position: center;
    background-size: 20px 20px;
    cursor: pointer;
    transition: background-color 0.2s ease;

    &:hover,
    &:focus {
      background-color: var(--header-btn-hover-bg);
    }

    &--mail {
      background-image: url('/icons/mail.svg');
    }

    &--logout {
      background-image: url('/icons/logout.svg');
    }
}

4. 규칙 설명

  1. 기본 클래스 + 모디파이어를 항상 함께 사용
  2. 크기, 배경색 등 공통 속성 변경 → header-action만 수정
  3. 아이콘 교체/추가 → 모디파이어 클래스만 추가
  4. 네스팅 최대 1단계 유지 (&--modifier)
  5. HTML 구조 변경 없이 스타일만으로 UI 유지보수 가능

5. 유지보수 예시

/* 전체 버튼 크기 변경 */
.header-action {
  width: 40px;
  height: 40px;
}

/* 메일 아이콘 변경 */
.header-action--mail {
  background-image: url('/icons/mail-new.svg');
}

6. 장점

  • 일관성: 동일한 구조/크기/패딩 유지
  • 확장성: 모디파이어만 추가하면 새 버튼 즉시 적용
  • 유지보수성: 공통 변경은 한 줄 수정으로 전체 반영
  • 가독성: 클래스 이름만 보고 역할과 기능이 명확
